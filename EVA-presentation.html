CHAPTER 02 · DATA ISOLATION
Your office. Your answers only.

Every authenticated user carries an office identity — either Albania or Serbia. The retrieval layer enforces a hard namespace boundary: queries are matched only against documents tagged to that user's office.

Albania Office
User in Tirana asks a question
Search is scoped to the ALB namespace. Albanian HR documents only. Serbian content is never retrieved, never surfaced.
Serbia Office
User in Belgrade asks the same question
Search is scoped to the SRB namespace. Serbian HR documents only. Completely independent from Albanian content.
Cross-office attempt
Can a Serbian user access Albanian documents?
No. The vector search filter is enforced at query time on the database level — there is no API surface to override the office scope. Answers are grounded only in the right country's materials.
How isolation is enforced — request flow
1
Spring Security resolves the authenticated user's office from their profile (mock or real)
2
Voyage AI embeds the user's query into a 1024-dim vector
3
pgvector cosine search is filtered by office = 'ALB' or office = 'SRB'
4
Claude Opus 4.8 receives only the retrieved office-scoped chunks as context and generates the answer
5
Audit log records every query, user, office, and cited sources for compliance tracking
Multilingual response
Detects & replies in your language
Ask in English, Albanian, or Serbian. Tika detects the input language and Claude replies in kind — no configuration needed.

CHAPTER 03 · KNOWLEDGE PIPELINE
From document to answer.
01
Source
Fetch documents
Ingestion job pulls HR documents from SharePoint (Albania via MS Graph API), a ZIP archive (Serbia), or a local folder. Configurable per office via environment variable.
02
Parse
Extract with Tika
Apache Tika 2.9 parses PDFs, PPTX, DOCX, and plain text into clean strings. Language is detected at this stage and stored as document metadata.
03
Chunk & Embed
Vectorize
Text chunks are sent to Voyage AI voyage-3 for 1024-dimension semantic embeddings. Each vector is tagged with office, source filename, and language.
04
Persist
Store in PostgreSQL
Embeddings land in a pgvector-enabled PostgreSQL instance running in Docker. The vector column enables cosine similarity search at query time without any external search engine.
Chat History
Persistent conversation context — per session, per user
Every message exchange is persisted to PostgreSQL, keyed by session ID and user. When the next message arrives, recent history is injected into the Claude prompt as context window — enabling follow-up questions, pronoun resolution, and coherent multi-turn conversations without repeating information.
Continuity — "What about the Serbia version?" works across turns
Faster UX — context is cached in DB, no re-fetch from SharePoint per turn
Auditability — full conversation trail for compliance review

CHAPTER 04 · WHAT SETS EVA APART
What makes EVA stand out.
Intelligence
Claude Opus 4.8 as the brain
The most capable Claude model drives every answer — not a lightweight summarizer. Tool-use agent architecture with configurable scope threshold ensures the LLM only answers when retrieval confidence is high enough.
Security
Hard office isolation at DB level
Cross-office data leakage is structurally impossible. The pgvector WHERE clause filter is enforced server-side, below the application layer — not just hidden in the UI.
Retrieval
Voyage AI semantic search
voyage-3 embeddings outperform generic models on multilingual domain text. Semantic similarity means users get correct answers even when phrasing doesn't match the document keywords.
Pipeline
Flexible ingestion sources
One ingestion runner handles SharePoint, ZIP, and local folders — configurable per office. Adding a new office or document source requires a single environment variable change.
Multilingual
EN · SQ · SR — detect & reply
Language detection at both ingestion time (Tika) and query time. The LLM receives language context in its system prompt and responds in whichever language the user wrote in.
Production-ready
Audit log + Docker + Spring Security
Every query, user, office, and cited source is logged. Full Docker Compose setup means zero-friction deployment. Spring Security provides a real auth layer ready to swap mock profiles for SSO.
